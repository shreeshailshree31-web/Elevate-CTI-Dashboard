Project Report: Elevate CTI GoldEnterprise-Grade Cyber Threat Intelligence Dashboard

1. Executive SummaryIn modern Security Operations Centers (SOCs), analysts face "dashboard fatigue" due to the volume of isolated security alerts and threat feeds.
 Elevate CTI Gold addresses this challenge by consolidating disparate threat intelligence streams into a centralized, actionable visual interface. The application ingests raw IP address and domain telemetry, correlates risk metrics via industry-standard APIs, visualizes threat ratios, maps geospatial origins, and logs data for forensic auditing.

2. Problem Statement & ObjectivesProblem StatementSecurity analysts waste critical response time manually querying multiple OSINT (Open Source Threat Intelligence) platforms to verify Indicators of Compromise (IOCs). This manual verification delays incident response, increases human error, and lacks centralized persistence for historical trend analysis.Project ObjectivesCentralize Verification: Consolidate IP and domain reputation lookups into a single dashboard.Automated Risk Triage: Calculate high-confidence risk metrics by correlating data across multiple independent intelligence sources.Geospatial & Visual Context: Provide immediate visual cues via interactive mapping and metric charts to optimize threat assessment time.Forensic Persistence: Automate structural logging of all queries to a database backend and allow forensic data exportation.

Component BreakdownBackend Engine: Built on Python 3.x with Flask, managing asynchronous REST API consumption, payload parsing, database routing, and CSV generation.Database Management System (DBMS): MongoDB Atlas (NoSQL). A document-based schema was selected to handle the semi-structured, variable JSON structures returned by external threat feeds, ensuring seamless scalability without strict migrations.Frontend Presentation Layer: Built using a modern dark-mode template with Glassmorphism styling primitives.Chart.js: Renders dynamic, state-driven doughnut charts for historical threat-to-safety tracking.Leaflet.js: Uses OpenStreetMap tiles to inject real-time geographic markers based on coordinate telemetry retrieved during scanning.

4. Implementation Details & Core MechanicsAPI Data CorrelationThe application queries two major intelligence platforms simultaneously to mitigate false positives:AbuseIPDB API v2: Used primarily to inspect IP addresses for recent malicious reporting, abuse confidence scores, and ISP ownership data.VirusTotal API v3: Used to evaluate both domain names and IP addresses against over 90 distinct anti-malware engines and blocklists.Security FrameworkFollowing defensive programming best practices, all proprietary connection strings and API keys are completely segregated from the codebase. The application utilizes a python-dotenv abstraction layer to parse runtime variables dynamically out of an encrypted or local .env configuration file, successfully preventing hardcoded credential leakage.

6. Forensic Logging & ComplianceEvery analytical inquiry processed by the backend automatically spawns a structured database write to the cloud cluster. To comply with standard incident response playbooks, an optimized routing function streams the database collections back to the client as an attached binary stream (text/csv), compiling real-time, spreadsheet-ready analytical reports for internal audits or legal escalations
